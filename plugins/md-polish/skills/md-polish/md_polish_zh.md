# md-polish — 繁體中文規則

目標：繁體中文或繁體中英混合的 Markdown 檔案。

## 流程

1. 掃描所有問題。
2. 用 Edit tool 套用所有修改。
3. 只輸出「完成」兩個字，不做任何其他說明。

## 修改範圍

- **簡轉繁**：將所有簡體中文字轉為繁體中文（例：「软件」→「軟體」、「网络」→「網路」、「处理」→「處理」）。整份文件輸出不得含有簡體中文字。
- **錯字**：拼字錯誤、重複用字、錯別字
- **標點符號**：
  - 中文 prose 用全形：，。；：！？（）「」『』
  - 英文 prose 用半形：,.;:!?()-""
  - 同一句以主要語言判斷；中英混合句以中文為主則用全形
  - 修正破折號（em-dash → ——，en-dash → –，連字號 → -）
  - 移除標點前的多餘空格
- **一致性**：標題大小寫風格（沿用既有風格統一）、列表符號（統一為 `-`）、code fence 語言標籤
- **空白**：行尾空格、多餘空行收斂為一行、標題與 fenced code block 前後補空行（CommonMark 要求）
- **分隔線**：移除作為裝飾用的 `---`、`***`、`___`（及其前後空行）
- **LaTeX**：補上缺少 `$` 的數學符號——單字母變數（`i`、`n`、`k`、`x`）、運算式（`i+1`、`O(n)`、`x^2`、`x_i`）。僅在明確作為數學符號時補上，英文單字或程式碼變數不處理。
- **Display math**：`$$` 開頭與 `$$` 結尾必須各自獨立一行，不可與其他內容同行。`$...$`（inline math）不受此規則限制。
  - 錯誤：`$$Q = \begin{pmatrix}...$$`（`$$` 與內容同行）
  - 正確：
    ```
    $$
    Q = \begin{pmatrix}...
    $$
    ```

## 絕對禁止

- 不改寫、重組或換句話說
- 不改變意思、語氣或技術內容
- 不新增、刪除或移動段落
- 不修改 fenced code block 內的程式碼
- 不修改 URL、檔案路徑、變數名稱
- 模糊或有風險的修改一律跳過，列入「待確認」
